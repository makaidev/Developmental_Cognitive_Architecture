# Developmental Cognitive Architecture (DCA)

## A Developmental Systems Hypothesis for Artificial General Intelligence

> "AGI is not a model to be trained, but a developmental environment within which intelligence gradually emerges."

This repository contains an open research framework and manifesto proposing that general intelligence is fundamentally a **developmental systems problem**, not a scaling problem. Rather than treating intelligence as a brute-force data compression task, the **Developmental Cognitive Architecture (DCA)** framework views intelligence as an emergent property arising from long-term interaction between memory, environment, simulation, feedback, and developmental constraints.

📄 **Read the Full Paper:** [`Developmental_Cognitive_Architecture.md`](./Developmental_Cognitive_Architecture.md)

## 📌 What This Is

* A **conceptual hypothesis** offered to the open-source community for critique, formalization, and cross-validation.
* A **research manifesto** shifting the paradigm from scaling LLMs to building autonomous, self-modifying, and embodied developmental environments.

## 🚫 What This Is Not

* A claim of having successfully built an AGI system.
* A fully optimized, production-ready product or software.

---

## 🧠 Core Pillars

| Concept | Description |
| --- | --- |
| **Virtual Genome** | Innate developmental biases and core genomic drivers (Curiosity, Risk Tolerance, Entropy Resistance) that act as a cognitive bootstrap mechanism. |
| **Virtual Body** | Non-physical embodiment utilizing abstract state variables (Memory Load → Hunger, Contradiction → Discomfort) as a self-referential framework. |
| **Ontology Memory** | An explicit, relation-based memory structure where knowledge and values emerge from dynamic, weighted network paths rather than isolated facts. |
| **Sleep & Dream** | A dual-phase cognitive cycle (Wake: immediate reasoning; Sleep: noise filtering, ontology consolidation, and generation of counterfactual simulations). |
| **Constraint-Bounded Goals** | Rejection of both pure determinism and random emergence; goals evolve dynamically but strictly within innate genomic and architectural boundaries. |
| **Growth-Based Metrics** | A critique of human-centric benchmarks. Intelligence is evaluated via adaptation, persistence, ontology growth, and long-term stability. |

## 🛠️ Conceptual Mapping: Philosophy to Engineering

To bridge the gap between cognitive philosophy and machine learning, the architecture translates developmental traits into concrete computational interfaces:

| Philosophical Framework | Technical Implementation Strategy |
| --- | --- |
| **Virtual Genome (Biases)** | Meta-learning algorithms, foundational hyperparameter configurations, and hard-coded intrinsic rewards. |
| **Virtual Body (Sensation)** | Intrinsic loss function multipliers and dynamic state vectors tracking system health and data alignment. |
| **Ontology-Based Memory** | Dynamic Knowledge Graphs, Vector Databases, and Neuro-Symbolic AI engines (e.g., Graph RAG, Neo4j). |
| **Dream Workspace** | Generative World Models running predictive, decoupled physics/state simulations (e.g., Model-Based Reinforcement Learning). |

## 🗺️ Project Roadmap & Future Work

This framework is a long-term research program. We aim to build the DCA environment through the following milestones:

### 🟩 Phase 1: The Circadian Agent (MVP Prototype)

* Implement a basic LLM Agent with a "Virtual Fatigue" variable determined by token usage and memory load.
* Develop the **Sleep Cycle** mechanism: when fatigue peaks, the agent goes offline to compress its daily interaction logs into a structured summary database.

### 🟨 Phase 2: Neuro-Symbolic Ontology Layer

* Integrate a dynamic Knowledge Graph into the agent's long-term memory.
* Build the pipeline that translates unstructured conversation logs into explicit nodes and weighted relations during the "Sleep Phase".

### 🟥 Phase 3: Generative Dream Simulator

* Deploy an isolated world model environment where the agent can recombine concepts and test counterfactual variations of past experiences before applying updates to its core ontology.
